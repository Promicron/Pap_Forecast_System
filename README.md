# SalesCast — AI Sales Forecasting & Decision Support System

End-to-end machine learning solution that forecasts retail sales trends and supports business decision-making. Built on the Kaggle Superstore dataset (10,194 transactions, 804 customers, Jan 2023 – Dec 2026). [Here](https://www.kaggle.com/datasets/himanshuuike/superstore-sales-dataset)

---

## Quick Start

### Option A — Docker Compose (recommended)

```bash
# 1. Download the Kaggle Superstore dataset
#    kaggle.com/datasets/vivek468/superstore-dataset-final
#    Place samplesuperstore.csv in the project root

# 2. Run the data pipeline to generate features + train models
docker compose run --rm api python pipeline/run_pipeline.py

# 3. Start the full stack
docker compose up --build

# API:       http://localhost:8000
# Swagger:   http://localhost:8000/docs
# Dashboard: http://localhost:5173
```

### Option B — Local development

```bash
# Python environment
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Run pipeline (ingest → clean → features → train → EDA)
python pipeline/run_pipeline.py

# Start API
uvicorn api.main:app --reload --port 8000

# Start frontend (separate terminal)
cd frontend && npm install && npm run dev
```

**Top XGBoost features:** `week_of_year`, `avg_discount`, `month`, `days_to_quarter_end`, `ewm_14d`

## Feature Engineering Highlights

- **55-column feature matrix** from 10,194 raw transactions
- **Temporal signals:** day-of-week, month, quarter, Fourier pairs (weekly + annual seasonality)
- **Lag features:** t-7, t-14, t-28, t-56 (no data leakage — all shifted by ≥1 day)
- **Rolling windows:** 7d / 14d / 28d / 90d mean + std
- **Business cycle:** days to quarter-end, US bank holiday proximity (Black Friday, Cyber Monday)
- **Exponentially weighted mean (EWM 14d)** for recent trend sensitivity

---

## Data Governance

- All customer PII (names, emails) SHA-256 hashed at ingestion — non-reversible
- Raw `Customer Name` and `Customer ID` fields dropped before any downstream processing
- `customer_id` is a `CUST_` prefixed 10-char hex hash
- Negative-margin transactions retained (real business signal) but flagged
- No raw PII in any saved artefact, model, or API response

## Extending the System

**Add a new model:** Implement `run(save=True) -> (model, cv_results, forecast_df)` in `pipeline/`, register the forecast CSV in `model_registry.py`, and add a router case in `routers/forecast.py`.

**Connect to Supabase:** Replace the CSV reads in `model_registry.py` with `supabase-py` queries. The schema maps directly: `daily_features` → Supabase table, `transactions_clean` → Supabase table.

**Deploy to Google Cloud Run:**
```bash
gcloud builds submit --tag gcr.io/PROJECT/salescast-api
gcloud run deploy salescast-api --image gcr.io/PROJECT/salescast-api --port 8000
```
